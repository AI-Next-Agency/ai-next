## Talking points

### 1. AI eğitim içeriklerimizin hukuki bağlayıcılığı / Türkiye'de AI eğitim regülasyonu

Soru: Müşterilere AI ve otomasyon hakkında anlattıklarımız bağlayıcı bir mevzuata mı dayanıyor, yoksa Türkiye'de AI eğitimi için halihazırda bir regülasyon var mı?

Ön araştırma notları (2026-04 itibarıyla):

- **Henüz doğrudan, yürürlükte bir "AI Kanunu" yok.** Türkiye'nin AB Yapay Zeka Tüzüğü'ne (EU AI Act) uyum süreci Orta Vadeli Program'da 2026 Q3 hedefi olarak duruyor. TBMM'ye 2024–2025'te sunulmuş üç ayrı kanun teklifi var (risk-temelli yaklaşım); henüz yasalaşmadı.
- **Yürürlükte olan dolaylı çerçeve KVKK (6698).** Kişisel veri işleyen her AI sistemi KVKK'ya tabi: VERBİS kaydı, işleme envanteri, güvenlik tedbirleri. KVKK 2026 taslak değişikliği AI eğitim veri setlerine (yasalılık, ayrımcılık yasağı) doğrudan değiniyor. KVKK 15 Nisan 2026'da "Agentic AI" rehberi yayımladı — müşteri sunumlarında referans verilebilir.
- **Eğitim sektörüne özel:**
  - MEB YEĞİTEK bünyesinde "Yapay Zekâ ve Büyük Veri Uygulamaları Daire Başkanlığı" kuruldu.
  - MEB "Eğitimde Yapay Zekâ Politika Belgesi ve Eylem Planı (2025–2029)" yayımlandı — sınıf içi AI kullanımı için operasyonel rehber.
  - YÖK Üretken Yapay Zekâ Etik Kılavuzu (2024): tezlerde/yayınlarda AI kullanımının beyanı zorunlu, AI yazarlığı yasak.
- **Bizim için pratik sonuç:**
  - "Bağlayıcı AI mevzuatı" diye sunmamalıyız — yanlış olur.
  - Doğru çerçeve: KVKK + sektörel rehberler (MEB/YÖK) + yaklaşan AB AI Act uyumu. Müşterileri "regülasyon gelmeden hazırlanın" mesajıyla konumlandırmak doğru.
  - Karar: slaytlarda/eğitim materyalinde tek bir "Regülasyon Durumu" sayfası ekleyelim, kaynaklar dipnotlu.

### 2. Anket otomasyon mekanizması — yapılanlar

Inflow Network AI Hazırlık Değerlendirmesi için uçtan uca otomasyon kuruldu. Form gönderiminden GitHub'a kayda kadar manuel müdahale yok.

**Mimari:**
```
Form (HTML, GitHub Pages)
   ↓ HTTPS POST
Cloudflare Worker (form-submission.js)
   ↓ repository_dispatch
GitHub Actions (process-inflow-assessment.yml)
   ↓ commit
projects/inflow-network/responses/*.md  +  INFLOW_NETWORK_ASSESSMENT_RESULTS.md (özet)
```

**Bileşenler:**
- **Form:** `projects/inflow-network/3-FORM_INDEX.html` — 13 adımlı, 11 değerlendirme sorusu, 1.0–5.0 AI Olgunluk Skoru otomatik hesaplama, mobil uyumlu, localStorage yedeği.
- **Backend:** Cloudflare Worker (`workers/form-submission.js`) — CORS kısıtlı (sadece izinli domain'ler), IP başına 5 dk rate limit (KV), input validasyonu, GitHub token Cloudflare Secrets'ta.
- **Otomasyon:** GitHub Actions workflow — payload'u parse eder, markdown response dosyası oluşturur, özet dosyasını günceller, commit atar.
- **Depolama:** Her cevap timestamped markdown olarak repo'da; özet dosyası agregat sonuçları tutuyor.

**Maliyet:** $0/ay (Cloudflare + GitHub free tier).
**Süre:** Form gönderiminden GitHub'da görünmeye kadar < 30 sn.
**Kapasite:** 1000+ eşzamanlı submission sorunsuz.

**Durum:** Production ready. Deployment checklist `DEPLOYMENT_CHECKLIST.md`'de. Test rehberi `E2E_TESTING_GUIDE.md`'de.

**Şu an bekleyen:**
- Cloudflare Worker deploy (KV namespace + GITHUB_TOKEN secret)
- Form URL'inin Inflow ekibiyle paylaşılması
- İlk gerçek submission ile end-to-end doğrulama

**Tartışılacaklar:**
- Bu şablonu Infinia ve diğer müşteriler için generic hale getirelim mi? (multi-tenant)
- Respondent'a otomatik teyit maili ekleyelim mi?
- Skor dağılımı için basit bir dashboard?
- Cevapların public repo'da durması — hassas müşteri verisi için private repo'ya geçiş düşünülmeli mi?
