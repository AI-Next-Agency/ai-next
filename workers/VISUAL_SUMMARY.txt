WORKER ENDPOINTS
================

POST /responses (200ms)
├─ Input: company, name, email, department, role, questions, aiMaturityScore, maturityLevel
├─ Process:
│  ├─ Validate fields
│  ├─ Rate limit check (5 per IP per 5 min)
│  ├─ D1 Transaction:
│  │  ├─ INSERT responses
│  │  ├─ UPDATE company_daily_metrics
│  │  ├─ UPDATE company_maturity_distribution
│  │  ├─ UPDATE company_score_distribution
│  │  ├─ UPDATE company_question_metrics
│  │  └─ UPDATE company_submission_timeline
│  ├─ Invalidate cache
│  └─ Trigger GitHub (optional)
└─ Response: { success, responseId, submittedAt, dashboardPreview }

GET /analytics/dashboard/:company (<50ms cold, <5ms warm)
├─ Query Parameters: none
├─ Process:
│  ├─ Check cache (Cloudflare, 5m TTL)
│  └─ If miss: Query 5 pre-agg tables in parallel
├─ Response: { summary, maturityDistribution, scoreDistribution, recentDailyMetrics, submissionTimeline }
└─ Headers: Cache-Control: public, max-age=300

GET /analytics/details/:company?breakdown=questions (30ms)
├─ Query Parameters: breakdown (all|maturity|questions|timeline)
├─ Process: Query company_question_metrics + optional joins
└─ Response: { questionBreakdown: [...] }

GET /responses/:responseId (10ms)
├─ Process: Primary key lookup in responses table
├─ Response: { id, company, name, email, questions, scores, submitted_at_iso, ... }
└─ Use: Audit trail, detail review


AGGREGATION FLOW (On Every Submit)
===================================

1. Response arrives: { company, name, score, questions, maturityLevel }
   |
2. Insert into responses table (source of truth)
   |
3. Update company_daily_metrics
   ├─ Count submissions for today
   ├─ Calculate avg_maturity_score
   └─ Parse & aggregate question responses
   |
4. Update company_maturity_distribution (all 5 levels)
   ├─ Count responses at each level
   ├─ Calculate percentage
   └─ Track 7-day trend
   |
5. Update company_score_distribution (8 buckets)
   ├─ Count responses in each bucket (1.0-1.5, 1.5-2.0, ...)
   └─ Calculate percentage
   |
6. Update company_question_metrics
   ├─ Parse question_responses JSON
   ├─ Calculate per-question averages
   └─ Build distribution (count at each score level)
   |
7. Update company_submission_timeline
   ├─ Increment daily bucket
   └─ Update avg_maturity_score for bucket
   |
8. Invalidate dashboard cache
   └─ Next GET /analytics/dashboard will recompute
   |
9. (Optional) Trigger GitHub Actions
   └─ For archival/reporting
   |
All complete in <200ms via D1 transaction


PERFORMANCE PROFILE
===================

Operation                  | P50    | P95     | Notes
---------------------------+--------+---------+-----------------------------------
POST /responses            | 180ms  | 250ms   | Include validation + aggregation
GET /dashboard (cold)      | 40ms   | 70ms    | Parallel queries on 5 tables
GET /dashboard (warm)      | 2ms    | 5ms     | Cloudflare Cache API
GET /details/:company      | 25ms   | 50ms    | Single pre-agg table
GET /responses/:id         | 8ms    | 15ms    | Primary key lookup

Scaling: O(1) for dashboard (pre-aggregates are fixed per company)
         O(n) for submit (n = # aggregations, typically 6-7)

QUICK REFERENCE: WHAT EACH TABLE IS FOR
========================================

responses
  └─ Raw form submissions (immutable log)
    └─ Query: "Get all responses for company X"
    └─ Query: "Get response details for audit"

company_daily_metrics
  └─ Daily snapshot (one row per day per company)
    └─ Query: "How many responses today? What's avg score?"
    └─ Use: Daily KPI cards, trend line

company_maturity_distribution
  └─ Level breakdown (one row per level per company)
    └─ Query: "What % are at each maturity level?"
    └─ Use: Pie chart, level breakdown table

company_score_distribution
  └─ Histogram buckets (8 rows per company)
    └─ Query: "Score distribution for chart?"
    └─ Use: Bar chart, histogram visualization

company_question_metrics
  └─ Per-question stats (one row per question per company)
    └─ Query: "Which question has lowest average?"
    └─ Use: Question breakdown heatmap, weak spots

company_submission_timeline
  └─ Time-bucketed activity (many rows per company)
    └─ Query: "Submission trend over last 30 days?"
    └─ Use: Line chart, activity timeline

aggregate_execution_log
  └─ Audit trail (for debugging)
    └─ Query: "Did aggregation fail? When?"
    └─ Use: Monitoring, troubleshooting
