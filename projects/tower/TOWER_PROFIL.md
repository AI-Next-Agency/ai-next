## 🚀 What They Do

### Core Product
Tower is a **Python-native data orchestration and deployment platform** designed for the "last mile" of data engineering. It unifies:
- **Python-native orchestration** for pipelines, agents, and workflows
- **Managed compute execution** (containerized in Kubernetes)
- **Iceberg-based analytical storage** (optional lakehouse format)

### Key Capabilities
- Towerfile configuration (no infrastructure management required)
- Integration with dbt, dltHub, Jupyter notebooks, Python agents
- **AI/ML ready**: OpenAI, Claude, LangChain, Ollama, HuggingFace support
- APIs for embedding as execution layer in third-party applications
- Version control and environment management
- Zero infrastructure operations

### Current Traction (Feb 2026)
- **200,000+** pipeline runs across **30,000+** unique apps
- **70,000** monthly SDK downloads
- Customers: Taktile (FinTech), Inflow, a-Gnostics (Energy), CosmoLaser

## 💻 Technology Stack

### Backend & Orchestration
- Kubernetes-based execution (containerized Python workloads)
- Python runtime with SDK support
- DAG-based workflow execution
- Multi-tenant control plane

### Storage & Data Format
- Apache Iceberg (primary analytical storage)
- Snowflake, Spark, and broad ecosystem compatibility
- Optional managed Iceberg storage

### Integrations
- dbt Core (dependencies, seed, build commands)
- dltHub data loading
- SQLMesh transformations
- Jupyter notebooks

- LLM/AI frameworks

### AI/ML Integration
- **OpenAI, GPT-4o, Claude 3** support
- **LangChain** framework integration
- Ollama (local LLMs)
- llama.cpp compatibility
- HuggingFace models

## 📊 Market Position & Competition

### Market Positioning
"**Databricks of the Python Era**" - specifically targeting Python developers for data orchestration and AI agent deployment.

### Competitive Landscape

**Direct Competitors:**
- Apache Airflow (mature, large community, infrastructure overhead)
- Prefect (modern, cloud-native, but not Python-focused)
- Dagster (developer-first, asset-aware, orchestration-focused)

**Indirect Competitors:**
- Databricks (broader platform)
- Snowflake (data warehouse focus)
- dbt Cloud (transformation only)

### Key Differentiators
1. **Python-first design** - Code-centric vs. configuration
2. **Managed infrastructure** - No Kubernetes/Spark management
3. **Open data formats** - Iceberg integration for multi-platform compatibility
4. **AI-ready** - Built-in LLM/agent support from ground up
5. **"Last-mile" focus** - Solves AI-generated code deployment
6. **Zero ops** - Full abstraction of compute layer

### Market Dynamics
- Growing "post-modern data stack" trend (composable, open formats)
- AI-generated code becoming mainstream (Claude, GPT)
- Trend away from monolithic platforms toward specialized tools
- Python dominance in data science and AI
- Data observability market growing 12.2% CAGR (2024-2030)

**Estimated TAM**: $2-4B (data orchestration subset of broader data management market)
